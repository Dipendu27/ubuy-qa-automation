Scope: assertion integrity and safety hardening only. No new features, no dependency
changes, no Docker/CI/visual-baseline changes (those are scoped to v2.0.2).

## Site observations recorded

- All 10 static-page URLs return **HTTP 403 to non-browser HTTP clients** (Cloudflare WAF) — consistent with the framework's headed-browser requirement. Real 200-vs-404 drift for these routes must be confirmed in a live headed run; no `fixme()` markers were planted on guesses.
- The store-switch endpoint could not be fingerprinted offline; the rewritten network spec discovers it at runtime via a URL predicate (`store|region|country|currency|switch`) and fails loudly if no switch-related traffic is observed. If a live run shows the switch is a pure page navigation, keep the response + cookie-diff assertions and re-annotate to `session-verified` per the task spec.

assumption, adapt it to assert the actual observed behavior (documented with URL + date)
— do not loosen it back to a tautology.
